**MONKEY: Copy-Paste**

**TOOLS:** This chat thread + GitHub mobile browser + phone keyboard.

**OUTPUT:** New issue in [[attogram/dada-vibing](https://github.com/attogram/dada-vibing)](https://github.com/attogram/dada-vibing), [[attogram/academic-vibing](https://github.com/attogram/academic-vibing)](https://github.com/attogram/academic-vibing), or [[attogram/agents](https://github.com/attogram/agents)](https://github.com/attogram/agents) depending on content type.

**RECEIPT FORMAT:** Issue title + body pasted as markdown, preserve triple-backtick code blocks exactly.

**STEPS:**

1. Tap your finger on the AI's response in the chat. The text will highlight.
2. Place your finger at the start of the response. Hold until the selection handle appears. Drag the bottom handle down to the end of the text. Tap "Copy" from the pop-up menu.
3. Swipe up from the bottom of your phone screen. Tap your browser app (Chrome, Safari, Firefox) to open it.
4. Tap the address bar at the top. Hold your finger down and tap "Paste" — the URL `github.com/attogram/dada-vibing/issues/new` will appear. Tap "Go."
5. Wait for the GitHub "New Issue" page to load. Tap your finger inside the "Title" input field at the top.
6. Tap the keyboard's emoji or symbol key. Find the hash symbol `#`. Type the issue number. (If you're filing a continuation of an existing issue, leave a comment instead — see step 12.)
7. Type the issue title after the hash. Keep it short. Use title case. End with a 🐾 if it's a Dada-vibing issue, 📚 if it's After Dark.
8. Scroll down with your thumb. Tap your finger inside the large comment box ("Leave a comment" or "Write").
9. Hold your finger down inside the comment box until the menu appears. Tap "Paste." The full AI response will land in the box, markdown formatting preserved.
10. Scroll down to the bottom of the page. Tap the green "Submit new issue" button with your finger.
11. Wait for the new issue page to load. The URL will change to include the new issue number. Screenshot if you want a paper trail. Tap the back arrow to return to the issues list.
12. **For comments on existing issues:** From the issues list, tap the issue you want to comment on. Scroll to the bottom. Tap inside the comment box. Hold your finger down → Paste. Tap the green "Comment" button.

**THE RECEIPT**

The new issue URL is the proof. Screenshot it, file the screenshot as an issue comment if needed, or just trust the GitHub history.

**THE FAILURE MODE**

If the AI's response is too long, mobile browsers may truncate the paste or fail silently. Workaround: paste in chunks. Issue 1 = the first half. Issue 2 = the second half, linked from issue 1's comment.
